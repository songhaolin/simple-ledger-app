## 💡 开发规范总结

### ✅ 已应用的最佳实践

1. **小步快跑**
   - 每个功能拆分成 3-5 个步骤
   - 每个步骤独立完成和提交
   - 避免大块操作

2. **主动汇报**
   - 开始前说明要做什么
   - 每完成一步立即汇报
   - 失败时也汇报原因

3. **失败即停**
   - 工具调用失败，立即汇报
   - 不继续后续操作
   - 说明已完成的工作

4. **TDD 严格遵循**
   - 先写测试（RED）
   - 再写代码（GREEN）
   - 按功能逐个开发

5. **问题修复**
   - Edit 工具匹配问题 → 使用 write 重写
   - 对话中断 → 任务拆分 + 进度汇报
